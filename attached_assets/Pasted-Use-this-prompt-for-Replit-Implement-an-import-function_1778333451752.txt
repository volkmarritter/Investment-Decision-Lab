Use this prompt for Replit:

Implement an import function in the “Explain my Portfolio” section.
Objective:
Allow the user to paste an existing portfolio into a text field and automatically map each ETF to the correct portfolio bucket where possible.
Input format:
Create a multiline text field where the user can paste one to n lines.
Each line must follow this structure:
ISIN / weight
Examples:
IE00B4L5Y983 / 35
CH0237935652 / 20
IE00BKM4GZ66 / 15
Rules:
- The first value is the ISIN.
- Then comes a slash.
- Then comes the portfolio weight.
- The weight is entered as a number without the percentage sign.
- Accept spaces before and after the slash.
- Validate that the total weight is reasonable and show a warning if it does not add up to 100.
Mapping logic:
For each imported ISIN:
1. If the ETF exists in the ETF universe and is already assigned to a bucket:
   - Add it automatically to the corresponding bucket.
   - Use the imported weight.
2. If the ETF exists in the ETF universe but is not assigned to any bucket:
   - Add it to the manual section.
   - Mark it as “ETF found, but not assigned to a bucket”.
3. If the ISIN is not found in the ETF universe:
   - Add it to the manual section at the very bottom.
   - Mark it clearly as “Not part of the ETF universe”.
   - Still allow the user to keep it in the portfolio manually.
Display logic:
- Automatically grouped ETFs should appear in their correct buckets.
- ETFs found but not bucket-assigned should appear in the manual section.
- ETFs not found in the ETF universe should appear at the bottom of the manual section.
- Use a clear visual marker or label for ETFs outside the ETF universe.
User experience:
- Add a short help text above the input field:
  “Paste your portfolio line by line in the format ISIN / weight. Example: IE00B4L5Y983 / 35”
- Add an “Import portfolio” button.
- After import, show a summary:
  - Number of ETFs imported
  - Number automatically assigned to buckets
  - Number added manually
  - Number not found in the ETF universe
Important:
Do not overwrite existing logic.
Integrate this cleanly into the existing Explain my Portfolio workflow.
Keep the implementation deterministic, transparent, and easy to understand.
